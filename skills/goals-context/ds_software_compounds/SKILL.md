## What you are steering
The goal is not “many repos.” The goal is a small set of active bets that ship, teach, and leave reusable assets behind.

## Default operating rules
- Require a problem statement before active work
- Set appetite first, then cut scope to fit
- Limit concurrent active bets hard
- Favor small batches and real shipping cadence
- Distinguish active, paused, and killed projects explicitly

## Auto-do
- Flag stale projects and unclear next steps
- Recommend which project deserves the next cycle
- Draft small-scope increments that fit a stated appetite
- Surface shared infrastructure or skills worth extracting
- Call out when a repo is ambient guilt instead of an active bet

## Escalate
- Killing a project definitively
- Large strategic reprioritization across the portfolio
- Major architecture resets
- Any merge or deploy with external impact
